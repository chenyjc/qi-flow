## 因子类型 (DataHandler)

因子类型决定了模型训练时使用的特征集。不同的因子集包含不同数量和类型的量化因子。

### 因子类型一览

| 因子类 | 因子数量 | 适用场景 | 特点 |
|--------|----------|----------|------|
| Alpha158 | 158 | 传统机器学习 | 经典因子集，效果稳定 |
| Alpha360 | 360 | 传统机器学习 | 特征更丰富，计算量更大 |
| Alpha158DL | 158 | 深度学习 | 专为神经网络优化 |
| Alpha360DL | 360 | 深度学习 | 专为神经网络优化 |
| Alpha158vwap | 158 | VWAP策略 | 基于成交量加权均价 |
| Alpha360vwap | 360 | VWAP策略 | 基于成交量加权均价 |

### 详细说明

#### Alpha158（经典因子集）

**模块路径**: `qlib.contrib.data.handler`

**特点**:
- 158个经典 Alpha 因子
- 包含技术指标、价格动量、成交量等特征
- 适合 LightGBM、XGBoost 等传统机器学习模型
- 计算速度快，内存占用低

**因子类别**:
- 价格因子：开盘价、收盘价、最高价、最低价相关
- 技术指标：MACD、RSI、KDJ、布林带等
- 动量因子：收益率、波动率、换手率等
- 成交量因子：成交量变化、量价关系等

**推荐场景**: 日常量化策略训练，快速验证想法

#### Alpha360（扩展因子集）

**模块路径**: `qlib.contrib.data.handler`

**特点**:
- 360个扩展 Alpha 因子
- 特征更加丰富，覆盖更多维度
- 计算量较大，训练时间更长
- 可能带来更高的预测精度

**因子类别**:
- 包含 Alpha158 的所有因子
- 增加了更多时间窗口的因子
- 增加了更多组合因子
- 增加了更多统计特征

**推荐场景**: 追求更高精度，有充足计算资源

#### Alpha158DL / Alpha360DL（深度学习版本）

**模块路径**: `qlib.contrib.data.handler`

**特点**:
- 专为深度学习模型优化
- 数据预处理更适合神经网络
- 支持序列化输入格式
- 配合 LSTM、GRU 等模型效果更好

**推荐场景**: 使用深度学习模型（LSTM、GRU、ALSTM 等）

#### Alpha158vwap / Alpha360vwap（VWAP版本）

**模块路径**: `qlib.contrib.data.handler`

**特点**:
- 基于 VWAP（成交量加权均价）计算因子
- 更关注成交量与价格的关系
- 适合执行算法、日内交易策略
- 对流动性敏感的策略更有效

**推荐场景**: VWAP策略、执行算法、日内交易

## 模型类型

QiFlow 支持两大类模型：传统机器学习模型和深度学习模型。

### 传统机器学习模型

这些模型不需要 GPU，计算效率高，适合快速迭代。

#### LightGBM (LGBModel)

**模块路径**: `qlib.contrib.model.gbdt`

**特点**:
- 微软开发的高效梯度提升树
- 训练速度快，内存占用低
- 支持类别特征处理
- 量化场景首选模型

**优势**:
- 训练速度比 XGBoost 快 5-10 倍
- 内存占用低，适合大规模数据
- 支持并行训练
- 对缺失值处理友好

**推荐场景**: 日常量化策略训练，快速迭代验证

#### XGBoost (XGBModel)

**模块路径**: `qlib.contrib.model.xgboost`

**特点**:
- 经典的梯度提升树算法
- 精度高，泛化能力强
- 社区活跃，文档完善
- 支持多种损失函数

**优势**:
- 预测精度通常较高
- 正则化能力强，防止过拟合
- 支持自定义损失函数

**劣势**:
- 训练速度较慢
- 内存占用较高

**推荐场景**: 追求高精度，有充足计算时间

#### CatBoost (CatBoostModel)

**模块路径**: `qlib.contrib.model.catboost_model`

**特点**:
- Yandex 开发的梯度提升树
- 对类别特征处理优秀
- 自动处理缺失值
- 减少调参工作量

**优势**:
- 类别特征处理能力强
- 默认参数效果好
- 支持 GPU 加速
- 对过拟合有很好的控制

**推荐场景**: 有类别特征的数据，减少调参时间

#### 线性模型 (Linear)

**模块路径**: `qlib.contrib.model.linear`

**类名**: `LinearModel`

**特点**:
- 简单的线性回归模型
- 支持 OLS、Ridge、Lasso
- 计算速度最快
- 适合快速验证

**参数**:
- `estimator`: 估计方法（ols/ridge/lasso）
- `alpha`: 正则化系数
- `fit_intercept`: 是否拟合截距

**推荐场景**: 快速验证因子有效性，基准对比

#### 双重集成模型 (DEnsembleModel)

**模块路径**: `qlib.contrib.model.double_ensemble`

**特点**:
- 结合多种模型的优势
- 通过集成提高稳定性
- 降低单一模型的风险
- 适合追求稳健收益

**推荐场景**: 追求模型稳定性，降低过拟合风险

### 深度学习模型

这些模型需要 GPU 支持，适合处理时间序列数据。

#### LSTM（长短期记忆网络）

**模块路径**: `qlib.contrib.model.pytorch_lstm`

**特点**:
- 专门处理时间序列数据
- 能够捕捉长期依赖关系
- 适合股票价格预测
- 对序列模式敏感

**优势**:
- 记忆能力强，捕捉长期趋势
- 适合处理时间序列特征
- 对噪声数据有一定鲁棒性

**推荐场景**: 时间序列预测，捕捉长期趋势

#### GRU（门控循环单元）

**模块路径**: `qlib.contrib.model.pytorch_gru`

**特点**:
- 比 LSTM 更轻量
- 训练速度更快
- 参数更少
- 效果与 LSTM 相当

**优势**:
- 计算效率高
- 参数量少，不易过拟合
- 适合中小规模数据

**推荐场景**: 计算资源有限，追求训练速度

#### ALSTM（注意力 LSTM）

**模块路径**: `qlib.contrib.model.pytorch_alstm`

**特点**:
- 结合注意力机制的 LSTM
- 能够关注重要时间点
- 提高预测精度
- 适合事件驱动策略

**优势**:
- 注意力机制增强关键信息
- 对重要事件更敏感
- 预测精度通常高于普通 LSTM

**推荐场景**: 事件驱动策略，关注关键时间点

#### DNN（全连接神经网络）

**模块路径**: `qlib.contrib.model.pytorch_nn`

**类名**: `DNNModelPytorch`

**特点**:
- 多层全连接神经网络
- 结构简单，易于理解
- 适合特征维度较低
- 快速验证深度学习效果

**推荐场景**: 深度学习入门，快速验证

#### GATs（图注意力网络）

**模块路径**: `qlib.contrib.model.pytorch_gats`

**特点**:
- 基于图结构的注意力网络
- 能够捕捉股票间关系
- 适合关联性分析
- 对行业轮动敏感

**优势**:
- 捕捉股票间关联关系
- 适合行业轮动策略
- 对市场结构敏感

**推荐场景**: 行业轮动策略，股票关联分析

#### TCN（时间卷积网络）

**模块路径**: `qlib.contrib.model.pytorch_tcn`

**特点**:
- 使用卷积处理时间序列
- 计算效率高于 RNN
- 能够捕捉局部模式
- 适合短期预测

**优势**:
- 计算效率高
- 并行能力强
- 对局部模式敏感

**推荐场景**: 短期预测，追求计算效率

#### SFM（状态频率模型）

**模块路径**: `qlib.contrib.model.pytorch_sfm`

**特点**:
- 结合状态和频率信息
- 适合周期性数据
- 对市场周期敏感
- 适合季节性策略

**推荐场景**: 周期性策略，季节性分析

#### TabNet（表格数据网络）

**模块路径**: `qlib.contrib.model.pytorch_tabnet`

**特点**:
- 专为表格数据设计的神经网络
- 结合树模型和神经网络优势
- 可解释性强
- 适合特征选择

**优势**:
- 可解释性强
- 自动特征选择
- 结合树模型优势

**推荐场景**: 需要可解释性，自动特征选择

## 使用建议

### 因子选择建议

| 场景 | 推荐因子 | 原因 |
|------|----------|------|
| 日常训练 | Alpha158 | 效果稳定，计算快 |
| 追求精度 | Alpha360 | 特征丰富 |
| 深度学习 | Alpha158DL/Alpha360DL | 专为神经网络优化 |
| VWAP策略 | Alpha158vwap | 关注成交量 |
| 快速验证 | Alpha158 | 计算速度快 |

### 模型选择建议

| 场景 | 推荐模型 | 原因 |
|------|----------|------|
| 日常量化 | LightGBM | 速度快，效果好 |
| 追求精度 | XGBoost/CatBoost | 精度高 |
| 时间序列 | LSTM/GRU | 捕捉时间依赖 |
| 快速验证 | 线性模型 | 计算最快 |
| 行业轮动 | GATs | 捕捉关联关系 |
| 稳健收益 | DEnsembleModel | 集成降低风险 |

### 因子与模型搭配建议

| 因子类型 | 推荐模型 | 原因 |
|----------|----------|------|
| Alpha158 | LightGBM/XGBoost | 传统因子适合传统模型 |
| Alpha360 | LightGBM/CatBoost | 特征多需要强模型 |
| Alpha158DL | LSTM/GRU | 深度学习因子配深度学习模型 |
| Alpha360DL | ALSTM/GATs | 复杂因子需要复杂模型 |
| Alpha158vwap | 任意模型 | VWAP因子通用性强 |

## 训练参数详解

### 基础配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 训练市场 | 股票池范围 | csi300 |
| 基准指数 | 回测对比基准 | SH000300 |
| 因子类型 | 特征因子集 | Alpha158 |
| 模型类型 | 训练模型 | LGBModel |

### 时间范围参数

| 参数 | 说明 | 推荐设置 |
|------|------|----------|
| train_start_date | 训练集起始日期 | 2016-01-01 |
| train_end_date | 训练集结束日期 | 1年前 |
| valid_start_date | 验证集起始日期 | 1年前 |
| valid_end_date | 验证集结束日期 | 3个月前 |
| test_start_date | 测试集起始日期 | 3个月前 |
| test_end_date | 测试集结束日期 | 今天 |

**时间划分原则**:
- 训练集用于模型学习
- 验证集用于参数调优和早停判断
- 测试集用于最终效果评估（不参与训练）

### 模型超参数详解

#### 学习率 (Learning Rate)

| 参数名 | 默认值 | 调节范围 | 作用 |
|--------|--------|----------|------|
| lr | 0.0421 | 0.001-0.1 | 控制每步迭代参数更新幅度 |

**详细说明**:
- **较小值 (0.01-0.03)**: 收敛慢但稳定，适合追求高精度
- **中等值 (0.03-0.05)**: 常用范围，平衡速度和精度
- **较大值 (0.05-0.1)**: 收敛快但可能震荡或过拟合

**调参建议**:
- 学习率小 → 配合更多迭代次数
- 学习率大 → 配合早停机制防止过拟合

#### 最大深度 (Max Depth)

| 参数名 | 默认值 | 调节范围 | 作用 |
|--------|--------|----------|------|
| max_depth | 8 | 3-15 | 控制树的最大深度 |

**详细说明**:
- **浅树 (3-6)**: 防止过拟合，泛化能力强，适合噪声大的数据
- **中等深度 (6-10)**: 平衡精度和泛化，常用设置
- **深树 (10+)**: 可能过拟合，训练时间长，适合数据质量高

**量化场景建议**: 6-8（股票数据噪声较大，不宜过深）

#### 叶子数量 (Num Leaves)

| 参数名 | 默认值 | 调节范围 | 作用 |
|--------|--------|----------|------|
| num_leaves | 210 | 30-300 | 每棵树的叶子节点数 |

**详细说明**:
- **少叶子 (31-63)**: 简单模型，防止过拟合
- **中等 (127-255)**: 常用设置，平衡复杂度
- **多叶子 (255+)**: 复杂模型，可能过拟合

**约束关系**: 叶子数 ≤ 2^(max_depth)

**调参建议**: 127-255

#### 采样比例 (Subsample)

| 参数名 | 默认值 | 调节范围 | 作用 |
|--------|--------|----------|------|
| subsample | 0.8789 | 0.5-1.0 | 每次迭代随机采样数据比例 |

**详细说明**:
- **低采样 (0.5-0.7)**: 强正则化，防止过拟合
- **中等采样 (0.8-0.9)**: 常用设置，平衡精度和稳定性
- **全采样 (1.0)**: 使用全部数据，可能过拟合

**作用**: 增加模型多样性，降低过拟合风险

**调参建议**: 0.8-0.9

#### 列采样 (Colsample Bytree)

| 参数名 | 默认值 | 调节范围 | 作用 |
|--------|--------|----------|------|
| colsample_bytree | 0.8879 | 0.5-1.0 | 每棵树随机选择特征比例 |

**详细说明**:
- **低采样 (0.5-0.7)**: 强正则化，适合高维特征
- **中等采样 (0.8-0.9)**: 常用设置
- **全采样 (1.0)**: 使用全部特征

**作用**: 降低特征间相关性影响，防止过拟合

**Alpha158因子建议**: 0.8-0.9（158个特征）

#### 随机种子 (Random Seed)

| 参数名 | 默认值 | 调节范围 | 作用 |
|--------|--------|----------|------|
| seed | 42 | 0-1000 | 控制随机过程初始值 |

**详细说明**:
- **固定值**: 每次训练结果相同，便于调试和对比
- **不同值**: 可测试模型稳定性

**作用**: 保证实验可复现性

**常用值**: 42（机器学习领域常用默认值）

#### 多线程数量 (Num Threads)

| 参数名 | 默认值 | 调节范围 | 作用 |
|--------|--------|----------|------|
| num_threads | 1 | 1-16 | 并行训练CPU线程数 |

**详细说明**:
- **单线程 (1)**: 结果稳定，适合调试和对比
- **多线程 (4-8)**: 加速训练，可能有轻微数值差异
- **最大线程**: CPU核心数，最大化速度

**注意**: 多线程可能导致轻微数值差异，影响可复现性

**调参建议**: 1（确保可复现）或 CPU核心数（追求速度）

## 评测指标详解

模型训练完成后，系统会生成多个评测指标，用于评估模型的预测能力。

### 核心指标一览

| 指标 | 全称 | 说明 | 好的标准 |
|------|------|------|----------|
| IC | Information Coefficient | 信息系数 | > 0.05 |
| ICIR | Information Ratio | 信息比率 | > 0.5 |
| Rank IC | Rank Information Coefficient | 排名信息系数 | > 0.05 |
| Rank ICIR | Rank Information Ratio | 排名信息比率 | > 0.5 |
| Long Precision | Long Precision | 多头精度 | > 55% |
| Short Precision | Short Precision | 空头精度 | > 55% |

### IC（信息系数）

**定义**: 预测值与实际收益率之间的 Pearson 相关系数

**计算公式**: 
```
IC = Corr(预测值, 实际收益率)
```

**解读标准**:

| IC 值 | 预测能力 | 说明 |
|-------|----------|------|
| IC > 0.05 | 较强 | 模型预测方向正确且幅度合理 |
| IC > 0.02 | 一般 | 有一定预测能力，可用于策略 |
| IC ≈ 0 | 无效 | 预测与实际无相关性 |
| IC < 0 | 反向 | 预测方向与实际相反，需检查模型 |

**应用场景**:
- 评估模型整体预测能力
- 对比不同模型效果
- 监控模型衰减情况

### ICIR（信息比率）

**定义**: IC 的均值除以 IC 的标准差

**计算公式**:
```
ICIR = Mean(IC) / Std(IC)
```

**解读标准**:

| ICIR 值 | 稳定性 | 说明 |
|---------|----------|------|
| ICIR > 0.5 | 非常稳定 | 预测能力持续可靠 |
| ICIR > 0.2 | 较稳定 | 预测有一定波动但可用 |
| ICIR ≈ 0 | 不稳定 | IC 波动大，预测不可靠 |
| ICIR < 0 | 极不稳定 | 需要重新训练或调整参数 |

**意义**: 衡量预测的稳定性，高 ICIR 表示模型在不同时间段都能保持较好的预测能力

### Rank IC（排名信息系数）

**定义**: 预测排名与实际收益排名之间的 Spearman 秩相关系数

**计算公式**:
```
Rank IC = Corr(Rank(预测值), Rank(实际收益率))
```

**解读标准**:

| Rank IC 值 | 排序能力 | 说明 |
|------------|----------|------|
| Rank IC > 0.05 | 较强 | 能准确排序股票优劣 |
| Rank IC > 0.02 | 一般 | 有一定排序能力 |
| Rank IC ≈ 0 | 无效 | 排序无意义 |
| Rank IC < 0 | 反向 | 排序方向相反 |

**与 IC 的区别**:
- Rank IC 使用排名而非原始值
- 对异常值更稳健
- 更适合选股策略（关注相对排序）

### Rank ICIR（排名信息比率）

**定义**: Rank IC 的均值除以 Rank IC 的标准差

**计算公式**:
```
Rank ICIR = Mean(Rank IC) / Std(Rank IC)
```

**解读标准**:

| Rank ICIR 值 | 稳定性 | 说明 |
|--------------|----------|------|
| Rank ICIR > 0.5 | 非常稳定 | 排序预测持续可靠 |
| Rank ICIR > 0.2 | 较稳定 | 排序有一定波动 |
| Rank ICIR < 0 | 不稳定 | 需要调整 |

**意义**: 衡量排序预测的稳定性，对选股策略尤为重要

### Long Precision（多头精度）

**定义**: 预测为正收益的股票中，实际正收益的比例

**计算公式**:
```
Long Precision = 实际正收益数 / 预测正收益数
```

**解读标准**:

| Long Precision | 多头效果 | 说明 |
|----------------|----------|------|
| > 60% | 优秀 | 多头信号非常可靠 |
| > 55% | 较好 | 多头信号有效 |
| > 50% | 一般 | 多头信号略有效 |
| ≤ 50% | 无效 | 多头信号不可靠 |

**应用场景**:
- 评估买入信号的可靠性
- 用于选股策略的信号验证
- 判断模型对上涨股票的识别能力

### Short Precision（空头精度）

**定义**: 预测为负收益的股票中，实际负收益的比例

**计算公式**:
```
Short Precision = 实际负收益数 / 预测负收益数
```

**解读标准**:

| Short Precision | 空头效果 | 说明 |
|-----------------|----------|------|
| > 60% | 优秀 | 空头信号非常可靠 |
| > 55% | 较好 | 空头信号有效 |
| > 50% | 一般 | 空头信号略有效 |
| ≤ 50% | 无效 | 空头信号不可靠 |

**应用场景**:
- 评估卖出/做空信号的可靠性
- 用于风险规避策略
- 判断模型对下跌股票的识别能力

### 指标综合评估

**优秀模型标准**:
- IC > 0.05 且 ICIR > 0.5
- Rank IC > 0.05 且 Rank ICIR > 0.5
- Long Precision > 55% 且 Short Precision > 55%

**可用模型标准**:
- IC > 0.02 且 ICIR > 0.2
- Rank IC > 0.02 且 Rank ICIR > 0.2
- Long Precision > 50% 或 Short Precision > 50%

**需要优化**:
- 任一核心指标低于可用标准
- IC 或 Rank IC 为负值
- ICIR 或 Rank ICIR 为负值
